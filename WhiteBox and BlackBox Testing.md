# Assignment 8 - Part B: White Box & Black Box Testing

## Testing Overview
 
### Definitions
 
**White Box Testing (Glass Box Testing)**
- Tester has full knowledge of internal code structure
- Tests internal logic, code paths, branches
- Focuses on code coverage, statement/branch coverage
- Requires programming knowledge
 
**Black Box Testing (Functional Testing)**
- Tester evaluates software without knowing internal code
- Tests functionality based on requirements
- Focuses on inputs and expected outputs
- No programming knowledge required
 
### Testing Framework
- **JUnit 5** - Unit testing framework
- **Mockito** - Mocking framework
- **Spring Boot Test** - Integration testing support
- **AssertJ** - Fluent assertions
 
---
 
## WHITE BOX TESTING
 
### Test Strategy
White box testing focuses on:
1. Statement Coverage
2. Branch Coverage
3. Path Coverage
4. Method Testing
5. Exception Handling
 
---
 
## BLACK BOX TESTING
 
### Test Strategy
Black box testing focuses on:
1. Functional Requirements
2. Input/Output Testing
3. Equivalence Partitioning
4. Boundary Value Analysis
5. Error Handling
